# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Key Commands

- `pnpm run typecheck` ‚Äî full typecheck across all packages
- `pnpm run build` ‚Äî typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` ‚Äî regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` ‚Äî push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` ‚Äî run API server locally
- `node_modules/.bin/tsx lib/db/src/seed.ts` ‚Äî re-seed restaurants/menu data

## Marsool App (Expo/React Native)

Arabic RTL delivery app. Features:
- Auth flow (phone + OTP + name)
- Home screen: real restaurant data from PostgreSQL API
- Restaurant detail + menu (real data)
- Order flow: free-text order ‚Üí real POST to API ‚Üí mock courier assigned ‚Üí status simulation
- Address management: CRUD via REST API, default address selection
- Search with filters, favorites, notifications, chat, order tracking

## Database Schema (`lib/db/src/schema/`)

- `restaurants` ‚Äî restaurant info (name, nameAr, category, rating, delivery details)
- `menu_items` ‚Äî menu items per restaurant (name, nameAr, price, image, category)
- `orders` ‚Äî orders with status (searching ‚Üí accepted ‚Üí on_way ‚Üí delivered), courier info
- `addresses` ‚Äî user delivery addresses with default flag
- `promo_codes` ‚Äî discount codes (percent/fixed, max_uses, expiry, isActive)
- `promo_uses` ‚Äî tracks per-user promo code redemptions (promoId, userId, orderId)

## API Endpoints (`artifacts/api-server/`)

- `GET /api/restaurants` ‚Äî list all restaurants (sorted by rating)
- `GET /api/restaurants/:id` ‚Äî restaurant details
- `GET /api/restaurants/:id/menu` ‚Äî restaurant menu items
- `GET /api/orders?userId=guest` ‚Äî list user orders
- `POST /api/orders` ‚Äî create order (auto-assigns mock courier)
- `GET /api/orders/:id` ‚Äî order details
- `PATCH /api/orders/:id/status` ‚Äî update order status
- `GET /api/addresses?userId=guest` ‚Äî list addresses
- `POST /api/addresses` ‚Äî create address
- `PUT /api/addresses/:id` ‚Äî update address
- `DELETE /api/addresses/:id` ‚Äî delete address
- `PATCH /api/addresses/:id/default` ‚Äî set default address
- `POST /api/orders/validate-promo` ‚Äî validate a promo code for the current user
- `GET /api/courier/earnings` ‚Äî courier earnings summary (today/week/total + recent deliveries)
- `GET /api/admin/promos` ‚Äî list all promo codes with use count
- `POST /api/admin/promos` ‚Äî create a promo code
- `PUT /api/admin/promos/:id` ‚Äî update a promo code
- `DELETE /api/admin/promos/:id` ‚Äî delete a promo code

## API Client

Generated by Orval from `lib/api-spec/openapi.yaml`. React Query hooks in `lib/api-client-react/`.
Expo app connects via `EXPO_PUBLIC_API_HOST` env var (see `artifacts/marsool/.env`).

## i18n

Arabic (default, RTL) + English (LTR). Keys in `artifacts/marsool/i18n/ar.json` and `en.json`.
All screens use `AppText` wrapper instead of React Native `Text`.
