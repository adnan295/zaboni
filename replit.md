# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Key Commands

- `pnpm run typecheck` вЂ” full typecheck across all packages
- `pnpm run build` вЂ” typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` вЂ” regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` вЂ” push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` вЂ” run API server locally
- `node_modules/.bin/tsx lib/db/src/seed.ts` вЂ” re-seed restaurants/menu data

## Marsool App (Expo/React Native)

Arabic RTL delivery app. Features:
- Auth flow (phone + OTP + name)
- Home screen: real restaurant data from PostgreSQL API
- Restaurant detail + menu (real data)
- Order flow: free-text order в†’ real POST to API в†’ mock courier assigned в†’ status simulation
- Address management: CRUD via REST API, default address selection
- Search with filters, favorites, notifications, chat, order tracking

## Database Schema (`lib/db/src/schema/`)

- `restaurants` вЂ” restaurant info (name, nameAr, category, rating, delivery details)
- `menu_items` вЂ” menu items per restaurant (name, nameAr, price, image, category)
- `orders` вЂ” orders with status (searching в†’ accepted в†’ on_way в†’ delivered), courier info
- `addresses` вЂ” user delivery addresses with default flag
- `promo_codes` вЂ” discount codes (percent/fixed, max_uses, expiry, isActive)
- `promo_uses` вЂ” tracks per-user promo code redemptions (promoId, userId, orderId)
- `restaurant_hours` вЂ” weekly schedules per restaurant (dayOfWeek 0-6, openTime, closeTime, isClosed)
- `notification_logs` вЂ” broadcast push notification history (title, body, target, sentCount, failedCount)
- `delivery_zones` вЂ” zone-based delivery pricing (id, label, fromKm, toKm, fee, isActive); admin-managed; courier keeps 100% of fee
- `courier_applications` вЂ” courier join requests (userId, status: pending/approved/rejected, fullName, vehicleType, vehiclePlate, idNumber, notes, adminNote)

## Object Storage

Replit Object Storage is configured (bucket: `replit-objstore-55ff7d0a-e715-4d9a-b9aa-3d6d7ae80f97`).
- `POST /api/storage/uploads/request-url` вЂ” get presigned upload URL (public, no auth)
- `GET /api/storage/public-objects/*` вЂ” serve public assets
- `GET /api/storage/objects/*` вЂ” serve private objects

Admin panel uses `ImageUpload` component (`artifacts/admin/src/components/ImageUpload.tsx`) to upload images for restaurants and menu items. The component uses the two-step presigned URL flow: request URL в†’ PUT file directly to GCS. Uploaded images are served via `/api/storage/objects/uploads/{uuid}`.

Library: `lib/object-storage-web/` вЂ” `useUpload` hook and `ObjectUploader` component (Uppy-based).
Server: `artifacts/api-server/src/lib/objectStorage.ts` вЂ” `ObjectStorageService` class.

## API Endpoints (`artifacts/api-server/`)

- `GET /api/restaurants` вЂ” list all restaurants (sorted by rating)
- `GET /api/restaurants/:id` вЂ” restaurant details
- `GET /api/restaurants/:id/menu` вЂ” restaurant menu items
- `GET /api/orders?userId=guest` вЂ” list user orders
- `POST /api/orders` вЂ” create order (auto-assigns mock courier)
- `GET /api/orders/:id` вЂ” order details
- `PATCH /api/orders/:id/status` вЂ” update order status
- `GET /api/addresses?userId=guest` вЂ” list addresses
- `POST /api/addresses` вЂ” create address
- `PUT /api/addresses/:id` вЂ” update address
- `DELETE /api/addresses/:id` вЂ” delete address
- `PATCH /api/addresses/:id/default` вЂ” set default address
- `POST /api/orders/validate-promo` вЂ” validate a promo code for the current user
- `GET /api/courier/earnings` вЂ” courier earnings summary; uses orders.delivery_fee directly (fixed bug)
- `GET /api/delivery-fee-preview?lat=&lon=` вЂ” preview delivery fee by Haversine distance from Damascus center
- `GET /api/admin/promos` вЂ” list all promo codes with use count
- `POST /api/admin/promos` вЂ” create a promo code
- `PUT /api/admin/promos/:id` вЂ” update a promo code
- `DELETE /api/admin/promos/:id` вЂ” delete a promo code
- `GET /api/admin/delivery-zones` вЂ” list delivery zones sorted by fromKm
- `POST /api/admin/delivery-zones` вЂ” create a delivery zone
- `PUT /api/admin/delivery-zones/:id` вЂ” update a delivery zone
- `DELETE /api/admin/delivery-zones/:id` вЂ” delete a delivery zone
